# ADR-000003 — Certifications Only Where Genuinely Valuable

**Date:** May 2026
**Status:** Accepted
**Decider:** NikiDigitals
**Version:** GitHub — Technical Reference

---

## Context

A 5-year learning journey alongside a complex build project requires a clear policy on certifications — when to pursue them, which ones, and why. Without a policy, certification pursuit risks becoming either a vanity exercise or a distraction from building.

## Decision

Certifications are pursued only where they add genuine value — to knowledge depth, to portfolio credibility, or to career positioning as Finance Transformation Architect. Certifications are not collected for their own sake.

**Proposed certification pathway — confirmed per phase:**

| Certification | Phase | Value Justification |
|---|---|---|
| AZ-900 Azure Fundamentals | 0 | Cloud literacy baseline |
| GitHub Foundations | 0 | Daily tool throughout the project |
| DP-900 Data Fundamentals | 1 | Data is the core of LES |
| AZ-104 Azure Administrator | 1 | Infrastructure depth |
| AZ-204 Azure Developer | 2 | Development credibility |
| DP-600 Fabric Analytics Engineer | 3 | Intelligence layer |
| AI-900 Azure AI Fundamentals | 3 | Gateway to agent development |
| AI-102 Azure AI Engineer | 4 | Agent development |
| AZ-305 Solutions Architect Expert | 5 | Flagship — earned after building |

## Rationale

Each certification is evaluated at phase entry. If the phase content does not warrant the certification, it is deferred or dropped. The list is a proposal, not a commitment.

## Consequences

- No certification is started without confirming its value at that phase
- Certification study is integrated with the relevant build phase — not front-loaded
- The portfolio demonstrates capability through what was built, not through certificate count
