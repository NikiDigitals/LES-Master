# ADR-000005 — UK English for All Outputs

**Date:** May 2026
**Status:** Accepted
**Decider:** NikiDigitals
**Version:** GitHub — Technical Reference

---

## Context

The project owner is a native Dutch speaker. All professional and portfolio output requires a consistent language standard. A decision was required on which English variant to use across all written outputs.

## Decision

All written outputs — documentation, GitBook entries, LinkedIn articles, website content, code comments, and commit messages — are written in UK English.

## Rationale

- Professional consistency across all published work
- UK English is standard in European professional and academic contexts
- Consistency within a single variant prevents mixed spelling and terminology

## Consequences

- All spell-check tools and editors are set to UK English
- Common differences applied consistently: colour not color, organisation not organization, analyse not analyze
- Claude sessions produce UK English output by default
- Any contributor or collaborator must follow the same standard
