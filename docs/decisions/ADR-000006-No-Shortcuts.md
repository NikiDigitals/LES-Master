# ADR-000006 — No Shortcuts on Understanding

**Date:** May 2026
**Status:** Accepted
**Decider:** NikiDigitals
**Version:** GitHub — Technical Reference

---

## Context

LES is built by someone starting from zero technical knowledge. The path from beginner to expert over five years requires a learning philosophy — a governing principle for how knowledge is acquired and applied.

## Decision

No layer of LES is built before it is understood. Depth of understanding is the standard. Speed of delivery is never the deciding factor.

## Rationale

- A system built on shallow understanding accumulates technical debt that compounds over time
- The portfolio value of LES depends on the ability to explain and defend every decision — which requires genuine understanding
- The career target — Finance Transformation Architect — requires the ability to design and evaluate systems, not just operate them
- Understanding compounds: deep knowledge at Phase 0 makes every subsequent phase faster and more rigorous

## Consequences

- Learning sessions precede build sessions for every new concept or technology
- If something is not understood, it is not built — the session stops and the learning gap is addressed
- Documentation captures the understanding, not just the output
- This principle applies to every layer — cloud infrastructure, data modelling, development, AI agents, security
